# Image Hub — 思源图床

一个思源笔记图床插件，自动将文档中的本地图片上传到自定义图床（阿里云 OSS / 腾讯云 COS），上传成功后替换为远程链接，上传失败的保留本地链接。

**简体中文** | [English](README.md)

## 快速上手

1. 打开 **设置 → 集市 → 已下载**，启用 **Image Hub**。
2. 点击编辑器顶栏的 **图床** 按钮（🖼️ 图片图标），打开图床管理。
3. 切换到 **存储配置** 标签页，选择提供商（阿里云 OSS / 腾讯云 COS），填写账号信息并保存、启用。
4. 回到 **当前文章** 标签页，点击 **替换当前文章中的本地图片**，即可一键上传。

## 功能

| 功能 | 说明 |
|------|------|
| **图床管理器** | 统一的对话框管理图片与配置，左侧导航切换视图 |
| **批量上传** | 一键上传当前文档全部本地图片 |
| **单张上传** | 逐张选择图片单独上传 |
| **覆盖上传** | 对已上传的远程图片，重新选择本地文件覆盖远端对象 |
| **多配置** | 新增、复制、切换、删除、启用/禁用多组存储配置 |
| **粘贴自动上传** | 粘贴图片后自动上传并替换（默认开启） |
| **切换自动上传** | 切换文档时自动上传本地图片（默认关闭） |
| **自定义路径** | 模板变量控制对象键路径 |
| **自定义域名** | 替换链接使用自己的 CDN 或域名 |
| **命令面板** | 从思源命令面板执行操作，无需打开顶栏 |

### 已支持的提供商

| 提供商 | 状态 |
|--------|------|
| 阿里云 OSS | ✅ 已支持 |
| 腾讯云 COS | ✅ 已支持 |
| 七牛云 | 📋 计划中 |
| S3 兼容存储 | 📋 计划中 |

> 仅上传并替换文档 `assets/` 下的本地图片；挂件/插件资源（如 `kmind/kmind.svg`）不会被当作本地图处理。

## 使用方式

### 打开图床管理器

三种入口：

1. 点击编辑器顶栏的 **图床** 按钮（图片图标）。
2. **设置 → 集市 → 已下载 → Image Hub → 打开图床管理**。
3. 命令面板执行 **Open Image Bed Manager**。

### 存储配置

在 **存储配置** 标签页填写和管理连接信息：

| 字段 | 说明 |
|------|------|
| 配置名称 | 显示用名称，多配置时便于区分 |
| 上传方式 | 阿里云 OSS / 腾讯云 COS |
| AccessKey ID | 用于签名上传请求 |
| AccessKey Secret | **仅保存在本地插件设置中**，不会发送到别处 |
| Bucket | 目标 Bucket 名称 |
| Endpoint | OSS 例如 `oss-cn-hangzhou.aliyuncs.com`；COS 例如 `cos.ap-guangzhou.myqcloud.com` 或 `ap-guangzhou` |
| 对象路径模板 | 默认 `siyuan/{docId}/{filename}` |
| 自定义访问域名 | 可选，例如 `https://img.example.com` |
| 粘贴图片后自动上传 | 开启后粘贴图片即上传并替换 |
| 切换文档时自动上传 | 开启后切换文档自动上传本地图片 |

支持多组配置；同一时间仅一组配置可启用。从页面顶部切换当前配置。

#### 对象键模板变量

| 变量 | 说明 |
|------|------|
| `{docId}` | 当前文档块 ID |
| `{filename}` | 原文件名 |
| `{name}` | 不含扩展名的文件名 |
| `{ext}` | 文件扩展名 |
| `{yyyy}` `{mm}` `{dd}` | 当前日期 |

### 图片操作

在 **当前文章** 标签页：

1. **替换当前文章中的本地图片** — 自动扫描文档，逐张上传全部本地图片。
2. **上传并替换** — 单张图片逐个手动上传。
3. **再次上传覆盖** — 对已管理的远程图片，选择本地文件重新上传覆盖远端对象。

上传完成后，成功替换的链接变为远程 URL，失败的保持本地链接不变。页面上方会显示汇总结果。

### 命令面板

| 命令 | 操作 |
|------|------|
| Open Image Bed Manager | 打开图床管理器 |
| Upload Current Document Local Images | 替换当前文档中的本地图片 |

## 安装

### 从集市安装

打开 **设置 → 集市 → 插件**，搜索 **Image Hub** 安装。

### 手动安装

1. 从 [Releases](https://github.com/jeefeng/siyuan-plugin-image-hub/releases) 下载最新 `package.zip`。
2. 解压到 `{工作空间}/data/plugins/siyuan-plugin-image-hub/`。
3. 重启思源，在 **设置 → 集市 → 已下载** 中启用插件。

> 插件目录名必须与 `plugin.json` 的 `name` 一致：`siyuan-plugin-image-hub`。

## 云存储配置

插件使用浏览器端 `PUT Object` 请求上传。Bucket 必须允许思源客户端的跨域请求。

### CORS 规则建议

| 设置 | 推荐值 |
|------|--------|
| 允许来源 | 实际的思源桌面端或浏览器端来源（自用可适当放宽） |
| 允许方法 | `PUT`、`GET`、`HEAD` |
| 允许 Header | `Authorization`、`Content-Type`、`x-oss-date`（OSS） |
| 暴露 Header | `ETag` |

### 安全建议

- 使用权限受限的子账号 AccessKey，仅授予目标 Bucket 必要的读写对象权限。
- AccessKey Secret **仅保存在本地插件设置中**，不会被发送到插件以外的任何地方。

## 推荐首次测试

1. 在 Bucket 配好 CORS。
2. 打开 Image Hub，在 **存储配置** 中保存并启用一组配置。
3. 新建一篇测试文档，插入一张本地图片。
4. 打开图床管理器，确认图片出现在 **当前文章** 中。
5. 点击 **替换当前文章中的本地图片** 或单张 **上传并替换**。
6. 上传成功后文档链接变为远程 URL；失败则链接不变。
